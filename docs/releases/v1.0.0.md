# baobab-probability-core 1.0.0

Première **release GitHub** du noyau : probabilités discrètes, simulations reproductibles et comparaison théorie / observations, sans dépendance scientifique obligatoire en production.

## Prérequis

- **Python 3.11+**

## Fonctionnalités

- **Combinatoire** : factorielles, permutations, arrangements, combinaisons (validations strictes sur les entiers).
- **Probabilité** : événements et espaces finis avec issues **hashables** (`str`, entiers, tuples, énumérations, etc.), union / intersection / conditionnement, Bayes, test d’indépendance (tolérance flottante).
- **Distributions** : Bernoulli, binomiale, géométrique, hypergéométrique, Poisson.
- **Simulation** : configuration à graine fixe, `SimulationResult` typé, simulateurs Bernoulli, binomiale, dé, urne (avec / sans remise).
- **Statistiques empiriques** : observations, fréquences, moments, descriptifs.
- **Comparaison** : écart probabilité / fréquence, adéquation (χ² de Pearson, variation totale).

## Qualité

- Typage strict (`mypy --strict`), `py.typed`, suite de tests avec **couverture ≥ 90 %** (seuil dans `pyproject.toml`, échec CI si non atteint).
- **CI GitHub Actions** : même enchaînement d’outils sur chaque **`push`** et **`pull_request`** (Python 3.11 sur Ubuntu).
- Outillage local documenté : black, flake8 (via Flake8-pyproject), pylint, mypy, bandit, pytest ; synthèse dans [`docs/release_validation.md`](https://github.com/baobabgit/baobab-probability-core/blob/main/docs/release_validation.md).

## Installation

Depuis le dépôt (développement) :

```bash
pip install -e .
```

Avec les dépendances de développement :

```bash
pip install -e ".[dev]"
```

## Compatibilité / ruptures mineures à noter

- `SimulationResult` : ne plus instancier avec `data=dict` ; utiliser les champs typés ou la propriété `data` en lecture seule après passage par les simulateurs du paquet.
